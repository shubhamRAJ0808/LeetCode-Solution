# LeetCode-Solution 💡

My solutions to LeetCode problems, written in Java. This repository is **automatically synced** using [LeetSync](https://github.com/LeetSync) every time I submit an accepted solution on LeetCode — so it stays up to date without manual uploads.

## 📌 About

Each folder represents one LeetCode problem, containing my accepted Java solution along with runtime and memory stats captured at submission time. This repo is both a practice log and a quick reference for revisiting problems before interviews.

## 📂 Folder Structure

Each problem gets its own folder, named as `<problem-number>-<problem-slug>`:

```
LeetCode-Solution/
├── 1-two-sum/
├── 20-valid-parentheses/
├── 33-search-in-rotated-sorted-array/
├── 153-find-minimum-in-rotated-sorted-array/
└── ...more added automatically with each submission
```

Every folder typically includes:
- ✅ The Java solution file
- ⏱️ Runtime (ms) and percentile vs other submissions
- 💾 Memory used (MB) and percentile vs other submissions

## 🔄 How This Repo Updates

I don't manually push code here. The flow is:

1. Solve a problem on [LeetCode](https://leetcode.com)
2. Submit and get an **Accepted** result
3. **LeetSync** automatically pushes the solution + stats to this repo

This means the commit history itself is a real-time log of my problem-solving activity.
